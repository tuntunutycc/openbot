# Plan - Batch Variations (Phase 7)

## Technical Approach
1. Extend catalog pipeline with a batch mode entrypoint:
   - `run_catalog_batch_pipeline(image_bytes, filename, instruction) -> BatchResult`
2. Reuse existing cutout stage once, then fan out into 3 variation edit calls.
3. Generate variation prompts from Claude as strict JSON array with exactly three items.
4. Render each variation via Photoroom `/v2/edit` with style/prompt-specific params.
5. Send successful images back as one Telegram album using `send_media_group`.

## Prompt Generation Strategy (Claude)
- Add helper module/function (e.g., in `services/catalog_pipeline.py` or new `services/catalog_variations.py`) to request:
  - JSON array length = 3
  - each item includes fields like:
    - `style_name`
    - `background_prompt`
    - optional `color_bias` / `layout_hint`
- Validate:
  - exactly 3 entries
  - non-empty prompts
  - fallback to deterministic local defaults if JSON invalid

## Photoroom Execution Strategy
- Option A (preferred): run 3 requests concurrently with safe limits (thread pool/async bounded concurrency).
- Option B (safe baseline): sequential calls if complexity/rate limits make concurrency risky.
- Each call should be isolated:
  - one failure must not invalidate the other successful variations.
- Use existing fallback behavior in Photoroom edit client for quota/billing:
  - if 402/403, return cutout fallback image instead of raising fatal error.

## Telegram Delivery Strategy
- Build `InputMediaPhoto` list from generated bytes.
- Send with `bot.send_media_group(chat_id, media=[...])`.
- If only one image available (full fallback path), use `send_photo` with explanatory text.
- If 2/3 successful, send available set and include note about partial fallback.

## Fallback Rules (mandatory)
1. Claude variation prompt generation fails:
   - use hardcoded 3 default prompts (minimalist/premium/vibrant).
2. Photoroom quota/billing for one or more variations:
   - substitute cutout fallback image for failed slots.
3. Photoroom quota/billing for all variations:
   - send one cutout fallback image with a clear message.
4. Never crash bot process due to one failed variant.

## Proposed Files
- `services/catalog_pipeline.py` (extend for batch orchestrator)
- `services/catalog_styles.py` (add variation defaults)
- `services/photoroom_edit_client.py` (if needed, variant-level helper return metadata)
- `bot.py` (catalog route updated to choose single or batch behavior)

## Non-Goals
- No UI for human selection beyond returning the album.
- No persistent storage of generated variants.
- No multi-page export workflow yet (reserved for future phase).
